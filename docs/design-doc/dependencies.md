# 外部ランタイム依存

本リポジトリのスキル・hookが必要とする外部ランタイム依存（依存パッケージ・汎用CLI・インタプリタなど）を、どう宣言・供給・検証するかの方針を定める。
基本となる方針（ホスト充足前提、明示宣言と再現性、再現的な操作のみ自律実行）は[設計原則](./principles.md)に定める。本節はその宣言の具体を扱う。

スキルとhookは依存の分類（依存パッケージ・外部ツール）を共有するが、宣言の場所と供給の仕方が異なる。

## 宣言の場所

| 成果物 | 宣言場所 |
| --- | --- |
| スキル | `SKILL.md`の`compatibility`フロントマターフィールドと本文の`## Requirements`節（内容を一致させる） |
| hook | 当該hookの`## Requirements`節（`.apm/hooks/README.md`） |

hookは`SKILL.md`を持たず、マニフェストが`.json`のため（[全体アーキテクチャ](./architecture.md)）、スキルの二層宣言（`compatibility`＋`## Requirements`）は適用しない。

## 宣言が必要なとき

外部依存（依存パッケージ、または外部ツール）があるときに宣言し、外部依存が無い成果物では省略する。
外部ツールへの依存はファイルに現れないため、ファイルの有無だけでは判定しない。
同梱した定義ファイル・ロックファイル・`scripts/`配下の言語ファイルに加え、成果物のロジックが呼び出す外部CLI・インタプリタ（`jj`・`gh`・`git`等）も依存として数える。

## 依存の分類

外部依存を「依存パッケージ」と「外部ツール」の2種に分類し、それぞれ別の規約で扱う。

### 依存パッケージ

プログラミング言語のライブラリ・パッケージとして、その言語のパッケージマネージャで導入するもの。
成果物に同梱した定義ファイルとロックファイルでバージョンを固定し、使用前にロックファイルから同期する。
具体例は textlintとそのプラグイン群、markdownlint-cli2、PyYAMLなど。

言語ごとのパッケージマネージャを次に統一する。

| 言語 | パッケージマネージャ | 依存定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| JavaScript / TypeScript | pnpm | `package.json` | `pnpm-lock.yaml` | `pnpm install --frozen-lockfile` |
| Python | uv | `pyproject.toml` | `uv.lock` | `uv sync --frozen` |

この表は、成果物が実行時に使う言語ライブラリを同梱して固定する場合に適用する（現状はスキル。hookは別方式をとる。後述）。
「配布時にインストール成功を保証」とは、配布された状態であれば記載のインストールコマンドを実行すれば必ず成功することを指す。自動インストールや成果物の同梱ではない。
新しい言語を同梱する場合は、その言語のパッケージマネージャを1つ選んで本表に追記する。

### 外部ツール

依存パッケージ以外の外部依存。
言語のパッケージマネージャで導入するライブラリではなく、汎用CLI・インタプリタ・パッケージマネージャ本体などのバイナリ。
具体例は gh、jq、jj、node、bun、pnpm、uvなど。

- バージョン要件とともに全列挙する。
- 使用前の存在確認コマンドを明記する。
- インストール方法は記載しない（入手方法は環境ごとに異なるため、成果物の責務としない）。
- 不在時の扱いは成果物で異なる（後述）。

## スキルの依存供給

スキルは依存を`compatibility`と`## Requirements`の2か所で宣言し、内容を一致させる。
依存が無いスキルでは両方を省略する。

依存パッケージはスキルに定義ファイルとロックファイルを同梱し、使用前にロックファイルから同期する。
`.venv`・`node_modules`などgit管理外のみを変更しgit検知の更新を生まないため、Agentが実行してよい。

```sh
# ロックファイルから依存を同期する（Agentが実行してよい）。
uv sync --frozen          # Python
pnpm install --frozen-lockfile  # JavaScript / TypeScript
```

外部ツールは存在確認のみ行う。

```sh
# 存在確認。欠けていれば停止・エスカレーション（Agentは導入しない）。
command -v gh >/dev/null 2>&1 || { echo "gh not found." >&2; exit 1; }
```

スキルは対話的に使われるため、外部ツールの不在や依存の追加・更新が必要な場合は、進行を停止しユーザーへエスカレーションする。
宣言のテンプレートは[スキル機構](./skills.md)に示す。

## hookの依存供給

hookは非対話で動く（[Hook機構](./hooks.md)）。供給はスキルと別の方針をとる。

- 外部コマンド呼び出しを最小限にし、設定・データ取得は実行環境の組み込み機能を優先して追加CLIに依存しない。
- 実行時に必要な言語パッケージは、プロジェクトのロックファイルに束ねず、版を固定して実行時に取得・キャッシュする。
- 依存は当該hookの`## Requirements`節に外部ツールとして列挙する。
- 不在・取得失敗時は停止せずfail-openで警告して通す（[Hook機構](./hooks.md)。導入はユーザーに委ねる）。

hookソースの開発ツーリング（lint・型チェック・テスト）は、配布物の実行時依存ではなく、実装言語の定義ファイルとロックファイルで`scripts/`配下に持つ。
