# スキル機構

スキルは、汎用のAI Agentにドメイン知識・ワークフロー・ツールを与えるモジュール式パッケージ。
`.apm/skills/<スキル名>/`に置き、`SKILL.md`を必須とする。
作成・検証の手順はskill-creatorスキルが提供する。本節では構成と設計上の規約を定める。

## スキルの種別

スキルは2種に大別され、どちらを作るかが設計判断を左右する。

- 能力拡張スキル: エージェントがまだ持たない知識やツールを与える。ドメイン固有のAPI・スキーマ・ワークフロー、同梱スクリプト、大きな参照ドキュメントなど。ドメインを網羅的に扱う必要がある。
- 嗜好エンコードスキル: 能力でなく振る舞いを調整する。コーディングスタイル・出力形式・語調や言語ルールなど。曖昧な好みは一貫しない挙動を生むため、正確で曖昧さのないルールを要する。

## 構成

スキルは`SKILL.md`と任意の同梱リソースからなる。

```text
<スキル名>/
├─ SKILL.md            # 必須。YAMLフロントマター + Markdownの指示
└─ 同梱リソース         # 任意
   ├─ scripts/         # 実行可能コード
   ├─ references/      # 必要時に読み込むドキュメント
   ├─ assets/          # 出力に使うファイル（テンプレート・画像など）
   └─ 依存ファイル       # 定義ファイル + ロックファイル（例: pyproject.toml + uv.lock）
```

`SKILL.md`のフロントマターは次のフィールドを持つ。

- `name`（必須）: ケバブケースの識別子。
- `description`（必須）: 何をするスキルでいつ使うか。エージェントがそのスキルを読み込むか判断する唯一の手がかりであり、起動条件を具体的に列挙する。
- `compatibility`（依存がある場合に必須）: 依存の1〜2行の機械可読な要約。本文の`## Requirements`節と一致させる。

同梱リソースは役割で分ける。

- `scripts/`: 決定的・反復的な操作のための実行可能コード。同期済み環境内で実行し、依存はインライン化せず定義ファイル+ロックファイルで宣言する。
- `references/`: 必要時にコンテキストへ読み込む詳細ドキュメント。`SKILL.md`を簡潔に保つために分割する。
- `assets/`: 出力に使い、コンテキストには読み込まないファイル。

## 段階的開示

スキルは3段階の読み込みでコンテキストを効率管理する。

1. メタデータ（`name` + `description`）: 常にコンテキスト内。
2. `SKILL.md`本文: スキル起動時。
3. 同梱リソース: 必要に応じて。

`SKILL.md`が肥大化したら`references/`へ分割し、各ファイルをいつ読むか`SKILL.md`から明示する。
コンテキストはシステムプロンプト・会話履歴・他スキルのメタデータと共有されるため、エージェントがまだ持たない情報だけを加える。

## 依存宣言

スキルの外部ランタイム依存は、`compatibility`フロントマターフィールドと本文の`## Requirements`節の2か所で宣言し、内容を一致させる。
依存が無いスキルでは両方を省略する。
依存の分類・供給の規約は[外部ランタイム依存](./dependencies.md)で定める。

宣言のテンプレートを次に示す。該当しない区分は省略してよい。

フロントマターの`compatibility`フィールドは次のとおり（1〜2行の要約）。

````markdown
compatibility: |
  Required: <インタプリタ・外部ツール>; packages: <パッケージ名>（<定義ファイル> / <ロックファイル> 経由）
  Scripts: <同梱スクリプト名>
````

本文の`## Requirements`節は次のとおり。

````markdown
## Requirements

使用前に依存パッケージをロックファイルから同期する（`uv sync --frozen`等。git管理外のみを変更するためAgentが実行してよい）。
依存の追加・更新、外部ツールの導入、代替手段へのフォールバックが必要な場合は、進行を停止しユーザーへエスカレーションする。

### 依存パッケージ

| 言語 | パッケージマネージャ | 定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| <言語> | <pm> | <定義ファイル> | <ロックファイル> | `<install cmd>` |

同期コマンドを次に示す。

```sh
<ロックファイルから同期するコマンド>
```

### 外部ツール

| ツール | バージョン要件 |
| --- | --- |
| <tool> | `>= x.y` |

存在確認コマンドを次に示す。

```sh
<command -v などによる存在確認>
```
````

## スクリプト設計

同梱スクリプトは非対話実行向けに設計する。

- 対話プロンプトを出さず、入力はフラグ・環境変数・stdinで受ける。
- `--help`でインターフェースを示す。
- データはstdout、診断はstderrへ。自由形式よりJSON/CSV/TSVを優先する。
- 冪等にし、再試行に耐える。
- 失敗種別ごとに異なる終了コードを使う。

コンパイル言語はプラットフォーム依存・配布の複雑さから非推奨とし、スクリプトはPython・Node.js/TypeScript・Bashを推奨する。
